# GitHub Publishing Setup

Use this checklist when creating the public GitHub repository.

## Repository Settings

Recommended values:

- Repository name: `AudioPilot`
- Description: `Tiny privacy-friendly macOS menu bar app that notifies you when the default audio output changes.`
- Visibility: Public
- Default branch: `main`
- License: MIT
- Topics:
  - `macos`
  - `swift`
  - `coreaudio`
  - `menu-bar`
  - `usernotifications`
  - `privacy`
  - `sandbox`
  - `open-source`

Recommended social preview image:

- `Resources/Assets.xcassets/AppIcon.appiconset/icon_512x512@2x.png`

## First Push

If this folder is not yet its own git repository:

```sh
git init
git branch -M main
git add .
git commit -m "Initial AudioPilot release"
git remote add origin git@github.com:YOUR_USERNAME/AudioPilot.git
git push -u origin main
```

If you already created the GitHub repository with a README, pull first and resolve conflicts before pushing.

## Branch Protection

After the first push, enable branch protection for `main`:

- Require pull request before merging.
- Require status checks to pass before merging.
- Require the `CI` workflow.
- Do not require signed commits unless all maintainers are ready for that workflow.
- Allow maintainers to bypass only for emergency release fixes.

## GitHub Features To Enable

- Issues
- Discussions
- Releases
- Private vulnerability reporting

Suggested Discussion categories:

- Announcements
- Q&A
- Ideas
- Show and tell

## Labels

Create the labels listed in [docs/LABELS.md](LABELS.md). They make the issue list feel organized from day one.

## Seed Issues

Open a few starter issues from [docs/INITIAL_ISSUES.md](INITIAL_ISSUES.md). This makes the project feel alive and gives visitors obvious ways to help.

Pin or highlight:

- macOS compatibility testing
- Signed/notarized release packaging
- Screenshot/GIF for README
- Localization ideas

## First Release

Use [docs/RELEASE.md](RELEASE.md) to prepare `v0.1.0`.

Attach:

- `AudioPilot-0.1.0-macos.dmg`
- SHA-256 checksum
- Source archive generated by GitHub

Do not present a locally ad-hoc signed DMG as a polished public release. For public users, prefer Developer ID signing and notarization.

## Repository Short Pitch

Use this in announcements:

```text
AudioPilot is a tiny macOS menu bar app for one very specific annoyance: knowing where your sound is going. It watches the default output device with public CoreAudio APIs and shows a native notification when it changes. No microphone access, no telemetry, no network.
```
