# Suggested GitHub Labels

Create these labels after publishing the repository.

| Label | Color | Purpose |
| --- | --- | --- |
| `bug` | `d73a4a` | Something broken or surprising. |
| `enhancement` | `a2eeef` | Small feature or improvement. |
| `documentation` | `0075ca` | README, docs, release notes. |
| `privacy` | `0e8a16` | Privacy, permissions, or security posture. |
| `security` | `b60205` | Security-sensitive work. |
| `release` | `5319e7` | Signing, notarization, DMG, GitHub Releases. |
| `compatibility` | `fbca04` | macOS version or hardware compatibility. |
| `design` | `c5def5` | Icon, menu, notification wording, visual polish. |
| `accessibility` | `1d76db` | Contrast, readability, VoiceOver, interaction quality. |
| `localization` | `bfdadc` | Translations and language behavior. |
| `good first issue` | `7057ff` | Friendly for new contributors. |
| `help wanted` | `008672` | Maintainers would welcome outside help. |
| `discussion` | `d4c5f9` | Needs conversation before implementation. |
| `wontfix` | `ffffff` | Intentionally outside project scope. |
